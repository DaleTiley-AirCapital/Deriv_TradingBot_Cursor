#102 - V2 Retirement, Legacy Code Purge, and V3-Only Active Path Lockdown

## What & Why
Tasks #101A through #101D are now complete.

All 4 active trading symbols have engine-native scoring:
- BOOM300
- CRASH300
- R_75
- R_100

That means the platform has crossed the threshold where the remaining legacy V2 logic, docs, labels, score paths, and dead compatibility code are now dangerous.

We already know from earlier work that old V2 concepts persisted far too long:
- old shared score labels
- old family language
- old generic scoring logic
- dead routes/pages/docs
- mixed docs causing AI agents to drift back toward V2 assumptions

This task is the controlled retirement of all remaining V2 code and documentation that is no longer part of the active V3 system.

This is NOT a fresh redesign task.
This is a **purge + lockdown** task:
- remove inactive V2 code
- remove stale V2 docs
- remove dead fallback paths
- keep only the active V3 truth
- ensure the app/server operate on V3 only

---

## Non-negotiable source of truth
The repo strategy source-of-truth remains binding.

The following must remain intact:
- only 4 active trading symbols:
  - CRASH300
  - BOOM300
  - R_75
  - R_100
- mode thresholds:
  - Paper = 85
  - Demo = 90
  - Real = 92
- per-symbol calibration
- no time-based forced exits
- no ATR-based TP/SL as primary exit logic
- no generic shared scoring as active engine scoring path
- no V2 family logic in the live path

Do NOT purge anything that contradicts the source-of-truth V3 system.

## Critical rules

### 1. Purge old code completely
If a V2 code path is no longer part of active V3 behavior:
- DELETE it
- do NOT comment it out
- do NOT keep a fallback
- do NOT keep bridge wrappers “just in case”

If something breaks after removal, fix it in V3.
Do not preserve legacy code for comfort.

### 2. Purge old docs completely
If docs still describe V2 concepts, delete or rewrite them.
Do NOT leave mixed docs where some sections are V2 and some are V3.

This is critical because AI agents are clearly being influenced by stale docs.

### 3. Do not break active V3 behavior
The purge must preserve:
- 4 active symbol-native engines
- current mode thresholds
- current UI pages
- current data/export pipeline
- current runtime state model

### 4. No fake cleanup
Do not just rename or hide V2 files.
Actually remove dead ownership and dead imports.

### 5. Fail loudly
If a route/page/module still depends on V2 code after removal, fix it immediately in the same task.
Do not silently restore old paths.

---

## Objective
Retire and purge all remaining V2 code, docs, labels, inactive scoring logic, and compatibility leftovers from the active system so that the repo becomes V3-only in all active paths.

This task must:
1. identify every remaining V2 artifact still in the active path or still capable of misleading the system
2. remove it cleanly
3. update docs so only V3 truth remains
4. verify that all live paths still work
5. leave the repo in a V3-only state
