# PART 5 — ENGINE DECISIONS VISIBILITY FIX

## T010 — Fix execution_status written for blocked/rejected signals
The Engine Decisions tab is currently showing no signals even though engines are scanning and generating rejected/blocked candidates.

Root cause 1:
When a signal is rejected before reaching allocator approval, the scheduler currently writes lifecycle state values like:
- `idle`
- `watch`
- `qualified`

into the `execution_status` column of `signal_log`.

That is wrong for UI visibility.

### Required fix
In the blocked/rejected signal logging path in `scheduler.ts`:
- keep lifecycle state passed to the candidate lifecycle manager exactly as-is
- but write `execution_status: 'blocked'` into `signal_log` instead of the lifecycle state

Lifecycle state is internal runtime state.
`execution_status` is UI-facing semantic outcome.

### Important
Do NOT change lifecycle machine semantics.
Only change what is persisted into the DB for signal visibility.

---

## T011 — Lower the default signal visibility threshold
Root cause 2:
The `signal_visibility_threshold` default is too high, so real rejected signals are hidden.

### Required fix
Change the default `signal_visibility_threshold` from:
- 75
to:
- 50

Update:
- the signals/decisions route default
- the startup/default platform_state seed value wherever this key is initialized

So a fresh environment immediately shows sub-75 blocked signals.

### Important
This is NOT a trading-threshold change.
It is only the UI visibility threshold for the Decisions tab.

Do NOT change:
- paper mode threshold
- demo mode threshold
- real mode threshold
- engine gates

---

## T012 — Extend signal visibility condition to include rejection_reason
Even with semantic `execution_status='blocked'`, the visibility query should also surface any signal that has a rejection reason.

### Required fix
In the signals/decisions route visibility query, add:
- `rejection_reason IS NOT NULL`

as an additional OR condition.

Result:
Any future signal that has a rejection reason should be visible in the Decisions tab, regardless of lifecycle state or raw score.
