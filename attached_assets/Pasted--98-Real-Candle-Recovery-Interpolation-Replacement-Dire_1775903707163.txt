#98 - Real Candle Recovery, Interpolation Replacement, Direct 1m Source Verification, and Final Canonical Truth Fix

## What & Why
Task #97 improved export truth and research ranking, but the system still has a critical unresolved problem:

- interpolated candles are still present in large numbers
- they are being excluded from signals/research, but they are not being actively replaced with real candles from the API
- we still do not have explicit proof whether live 1m candles are sourced directly from the API or derived from ticks
- the export UI still allows invalid date selection behavior even though backend range truth is improved

This task must fix the remaining backend truth issues before the final UI rebuild.

The key principle is:

### Interpolation is LAST RESORT ONLY
If a candle is missing:
1. fetch the real 1m candle from the API
2. write it into `candles`
3. replace any existing interpolated candle at that timestamp
4. only keep interpolation if the API truly cannot provide that candle

The `candles` table must remain the canonical truth source for:
- export
- research
- AI analysis
- feature generation
- live engines
- higher timeframe enrichment

The `ticks` table is for live tick context and monitoring, not the primary truth source for long-term canonical 1m history.
