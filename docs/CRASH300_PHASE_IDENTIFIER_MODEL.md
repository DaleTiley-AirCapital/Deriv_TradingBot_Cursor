# CRASH300 Phase Identifier Model

This document describes the deterministic CRASH300 research extraction layer used to measure what happens before, at, during, and after each detected CRASH300 move.

## Purpose

The phase identifier model is research-only. It does not tune trading behavior, does not call AI, and does not create trade signals by itself.

Its job is to extract repeatable identifiers from:
- detected CRASH300 moves
- M1 candle history
- CRASH300 parity output
- optional persisted backtest attribution

These identifiers form the evidence base for later decisions about:
- entry timing
- direction validation
- TP policy
- SL policy
- trailing activation
- trailing distance
- context invalidation

## Phase Groups

### Before move

Computed at:
- `T-240`
- `T-120`
- `T-60`
- `T-30`
- `T-15`
- `T-5`
- `T-1`

These are context-only identifiers and must not produce trades by themselves.

Included categories:
- trend persistence and upward drift
- crash-event recency and distance
- post-crash recovery quality
- volatility compression and expansion
- previous detected-move distance and prior move metadata

### Trigger

Computed at:
- `T-5`
- `T-3`
- `T-2`
- `T-1`
- `T0`
- `T+1`
- `T+3`
- `T+5`

These are trigger-only identifiers and are the only phase intended to justify entry timing later.

Included categories:
- candle structure
- 1/3/5 bar momentum
- momentum acceleration
- micro-break structure
- rejection and impulse strength
- trigger transition classification
- trigger direction and trigger strength

### During move

Computed from move start through:
- `+5`
- `+15`
- `+30`
- `+60`
- `+120`
- `+240`
- actual move end

Included categories:
- MFE / MAE
- favourable vs adverse sequencing
- bars to reach 25/50/75/100 percent of move
- pullback after favourable progress
- opposite impulse counts
- trailing evidence proxies
- realistic TP evidence proxies

These identifiers later inform:
- TP evidence
- SL evidence
- trailing activation evidence
- trailing distance evidence

### After move

Computed after move end through:
- `+5`
- `+15`
- `+30`
- `+60`
- `+120`
- `+240`

Included categories:
- reversal after move
- failed continuation
- post-move volatility and compression
- next-move direction and timing
- mean reversion speed

These identifiers later inform:
- context expiry
- invalidation timing
- re-entry blocking

## Context-only vs trigger-only

Context-only:
- trend persistence
- crash recency
- recovery quality
- compression / expansion structure
- previous move distance

Trigger-only:
- candle body and wick shape
- 1/3/5 bar returns and momentum
- momentum acceleration
- micro-break direction and strength
- rejection / impulse
- trigger transition
- trigger direction
- trigger strength

## Later policy use

These identifiers are intended to inform future policy layers:

### TP evidence
- `realisticTpPct`
- `realisticTpPctP50`
- `realisticTpPctP75`
- `realisticTpPctP90`
- `barsToMfe25/50/75/100`

### SL evidence
- `observedSafeInitialSlPct`
- `observedNormalMaePct`
- `observedP75MaePct`
- `observedP90MaePct`
- `oppositeImpulseMaxPct`

### Trailing evidence
- `observedTrailingActivationPct`
- `observedTrailingDistancePct`
- `observedMinHoldBarsBeforeTrailing`
- `maxPullbackBeforeContinuationPct`

### Context invalidation evidence
- `failedContinuationAfterMove`
- `reversalPctAfterMove`
- `barsToReversal25`
- `barsToReversal50`
- `barsUntilNextMove`
- `nextMoveDirection`

## Export endpoints

### Full report

`GET /api/calibration/runtime-model/CRASH300/phase-identifiers`

Query params:
- `startTs` optional
- `endTs` optional
- `limit` optional
- `includeMoves` optional, default `true`
- `includeAggregates` optional, default `true`

### Summary only

`GET /api/calibration/runtime-model/CRASH300/phase-identifiers/summary`

Returns aggregate statistics only.

### One move

`GET /api/calibration/runtime-model/CRASH300/phase-identifiers/move/:moveId`

Returns one move with its before, trigger, during, and after identifiers.

## Aggregate interpretation

Aggregate groups are generated by:
- direction
- move type
- runtime family
- selected bucket
- move size bucket
- quality tier
- eventual outcome
- optional backtest trade outcome
- combined keys such as direction plus move size bucket

For numeric identifiers the report computes:
- count
- min
- p10
- p25
- median
- mean
- p75
- p90
- max
- standard deviation

Trigger aggregates also include:
- transition counts at each offset
- trigger direction counts at each offset
- strongest trigger offset distribution
- first valid trigger offset distribution

Separation diagnostics rank which identifiers differ most across:
- direction
- move size buckets
- quality tiers
- runtime families
- linked backtest trade outcomes
